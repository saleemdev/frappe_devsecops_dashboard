### Updating Documentation
- When adding features, create feature-specific docs
- Update **IMPLEMENTATION_GUIDE.md** for user-facing changes
- Update **CHANGES_QUICK_REFERENCE.md** for quick lookups
- Archive old docs in `/docs/archive` if superseded

### Document Naming Convention
- `*_SUMMARY.md` - High-level overview
- `*_GUIDE.md` - Step-by-step instructions
- `*_IMPLEMENTATION.md` - Technical implementation details
- `*_CHECKLIST.md` - Verification/testing checklists
- `*_REVIEW.md` - Code/feature reviews
- `*_FIX.md` - Bug fix documentation
